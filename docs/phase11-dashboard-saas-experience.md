# Phase 11 - Dashboard and SaaS Experience

## Architecture

Phase 11 adds a separate React application under `frontend/` and a read-only
dashboard API layer under `/api`. The frontend does not reproduce repository
analysis. It visualizes the review history, risk intelligence, architecture
findings, smart-review output, and dependency graph already generated by the
backend workflow.

## Frontend Structure

- `src/pages`: login, dashboard, repositories, reviews, detail, analytics,
  architecture, dependency graph, timeline, and settings.
- `src/components`: shared states, tables, metrics, charts, and React Flow graph.
- `src/layouts`: responsive dashboard shell and sidebar.
- `src/api`: typed API clients for auth, reviews, repositories, and analytics.
- `src/hooks`: React Query hooks and cache keys.
- `src/store`: persisted Zustand auth and UI state.
- `src/routes`: protected application routing.
- `src/types`: dashboard API contracts.

## Authentication

1. The login page opens `GET /api/auth/github`.
2. The backend creates a short-lived OAuth state and redirects to GitHub.
3. GitHub redirects to `GET /api/auth/github/callback`.
4. The backend exchanges the code, loads the GitHub profile, signs an HMAC JWT,
   and redirects to `/auth/callback#token=...`.
5. The frontend stores the session through Zustand and sends it as a Bearer token.

When `DASHBOARD_DEMO_MODE=true` and OAuth is not configured, the same endpoint
creates a local demo session. This keeps the complete flow laptop-testable.

## API Routes

- `GET /api/reviews`
- `GET /api/reviews/:id`
- `GET /api/repositories`
- `GET /api/analytics`
- `GET /api/architecture`
- `GET /api/dependency-graph`
- `GET /api/user/profile`

Review history uses MongoDB when configured. Demo data is returned only when
dashboard demo mode is enabled.

## Data Model

New reviews persist dashboard-ready fields alongside the existing Phase 9
history document: PR title and URL, risk level, review mode, affected systems,
architecture findings, critical files, suggestions, hotspots, smart-review
output, changed files, and the serializable dependency graph.

## Run Locally

Backend:

```bash
npm run dev
```

Frontend:

```bash
npm run frontend:dev
```

Production frontend build:

```bash
npm run frontend:build
```

Use `frontend/.env.example` to configure a different API URL.

## Design System

The dashboard uses a dark-first engineering aesthetic with an accessible light
mode, responsive navigation, reusable loading and empty states, compact data
tables, Recharts analytics, and an interactive React Flow dependency explorer.
